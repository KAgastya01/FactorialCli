If you are reading this inside your local terminal or code editor right now, **congratulations!** You just completed the first hurdle of open-source development: fetching a remote repository onto your local system.

Let's get your hands dirty with your very first local compile before you change a single line of code.

---

### ⚡ Fast-Track Tutorial (Your First 120 Seconds)

Open your terminal inside this exact directory and run these three native, dependency-free commands:

#### 1. Compile the Core Engine
```bash
javac FactorialCli.java
```
*(This forces your local computer to convert the human-readable Java code into raw executable bytecode, generating a `FactorialCli.class` file).*

#### 2. Run your First Test Calculation
```bash
java -cp . FactorialCli -n 5
```
* **Expected Output:** `5! = 120`

#### 3. Test your Built-in Help System
```bash
java -cp . FactorialCli --help
```

### 🛠️ Your Next Steps

1. **Pick a Target**: Go to our online GitHub Wiki Roadmap or look at open repository Issues to see what features we need help adding.
2. **Create a Feature Branch**: Never code directly on `main`! Type:
   ```bash
   git checkout -b feature/my-cool-upgrade
   ```
3. **Break Things**: Tweak the code, run your compile command again, and see how the output changes.
4. **Push & Pull Request**: Push your branch to your online profile and submit it to see our automated GitHub Actions validation robot welcome you!
